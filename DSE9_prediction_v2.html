Prediction

Predictive Model คืออะไร

Predictive Model หรือ "โมเดลการทำนาย" คือ อัลกอริทึมทางคณิตศาสตร์ที่เรียนรู้จากข้อมูลในอดีต เพื่อคาดการณ์ผลลัพธ์ในอนาคตหรือเหตุการณ์ที่ยังไม่เกิดขึ้น โดยอาศัยเทคนิคทางสถิติและ Machine Learning สามารถช่วยตัดสินใจอย่างชาญฉลาด ลดความไม่แน่นอน และเพิ่มประสิทธิภาพในทุกอุตสาหกรรม

ตัวอย่างการใช้งานในชีวิตประจำวัน:

การเงิน

ธนาคารใช้โมเดลทำนายความเสี่ยงการผิดชำระหนี้ก่อนอนุมัติสินเชื่อ

สุขภาพ

แอปพลิเคชันสุขภาพทำนายความเสี่ยงโรคเบาหวานจากพฤติกรรมการใช้ชีวิต

การซื้อของออนไลน์

ระบบแนะนำสินค้า (Recommendation System) ทำนายว่าคุณอาจสนใจอะไรจากประวัติการซื้อ

การขนส่ง

แอปแผนที่ทำนายเวลาถึงจุดหมายโดยวิเคราะห์สภาพการจราจรแบบเรียลไทม์

การเกษตร

ทำนายผลผลิตพืชผลจากข้อมูลสภาพอากาศและดิน

Prediction ไม่ใช่ Forecasting!!! ถึงแม้ทั้งสองจะมีหน้าที่คือการพยากรณ์ผลลัพธ์ในอนาคตก็ตาม!!!

Prediction เป็นการคาดการณ์ผลลัพธ์โดยไม่จำเป็นต้องอ้างอิงเวลา ไม่ต้องสนใจข้อมูลอนุกรมเวลา แต่มุ่งเน้นที่ความสัมพันธ์ระหว่างตัวแปร (Features) มากกว่า

กลับกัน Forecasting จะเป็นการคาดการณ์ที่ต้องอ้างอิงเวลาเพื่อพยากรณ์สิ่งที่จะเกิดขึ้นในอนาคต

ML Life Cycle

การสร้าง Machine Learning (ML) โครงการหนึ่งต้องผ่านกระบวนการเป็นขั้นตอนที่ชัดเจน ตั้งแต่การกำหนดปัญหา (Scoping), การเตรียมข้อมูล (Data), การสร้างโมเดล (Modeling) ไปจนถึงการนำไปใช้งานจริง (Deployment) และการดูแลรักษา ขั้นตอนเหล่านี้ถูกเรียกรวมกันว่า ML Project Lifecycle ซึ่งแต่ละช่วงสามารถย้อนกลับมาทำได้เพื่อให้ได้ผลลัพธ์ที่มีประสิทธิภาพมากที่สุด รูปด้านล่างจะแสดงให้เห็นภาพรวมของกระบวนการทั้งหมดอย่างชัดเจน

ประเภทของ Predictive Model

ก่อนที่จะเรียนรู้โครงสร้างของโมเดลแต่ละตัวไปพร้อมๆกัน เรามาทำความเข้าใจกับประเภทโมเดลในจักรวาล Data Science กันดีกว่า โดยการแบ่งประเภทโมเดลใน Data Science นั้นสามารถทำได้หลายแบบและไม่มีการยึดหลักหลักใดหลักหนึ่ง เราได้ยกตัวอย่างการแบ่งประเภทโมเดลที่ขึ้นอยู่กับเกณฑ์ที่ใช้ ดังต่อไปนี้

เมื่อเรารู้จักประเภทของมันกันคร่าว ๆ แล้ว คราวนี้เรามาทำความรู้จักกับโมเดลในการทำนายกันเถอะ โดยเราจะแบ่งโมเดลออกเป็น 3 ประเภท อ้างอิงจากความแตกต่างของโครงสร้างโมเดลครับ เพราะการแบ่งดังกล่าวจะช่วยให้ผู้เริ่มต้นเรียนรู้เห็นภาพและความต่างของโมเดลในแต่ละประเภทได้ชัดเจนมากที่สุดนั่นเอง…

โมเดลพื้นฐาน : Linear Based Model

จากการเรียนรู้จาก Infographic คร่าว ๆ เราสามารถอธิบายเพิ่มเติมได้ว่าโมเดลประเภทนี้เป็นโมเดลที่มองหาความสัมพันธ์เชิงเส้นระหว่างตัวแปร โดยสมมติว่าข้อมูลสามารถอธิบายได้ด้วยเส้นตรงหรือระนาบในมิติที่สูงขึ้น โมเดลพวกนี้ง่ายต่อการตีความ และทำงานได้ดีถ้าความสัมพันธ์ของข้อมูลเป็นเส้นตรงจริง ๆ

Logistic Regression

โมเดลประเภท Linear Based ที่ถูกนำมาใช้ในการทำนายค่าคลาส (classification) ของข้อมูล เช่น ทำนายว่าลูกค้าพอใจหรือไม่ ทำนายว่าอีเมลจะเป็นสแปมหรือไม่ หรือ ทำนายว่าแพทย์จะวินิจฉัยผู้ป่วยเป็นโรคหรือไม่

ประเภทแยกย่อย:

Binary Logistic Regression

ใช้เมื่อมีผลลัพธ์แค่ 2 คลาส (Dichotomous Variable) เช่น Yes/No, Male/Female, Pass/Fail, 1/0, ป่วย/ไม่ป่วย, ตั้งครรภ์/ไม่ตั้งครรภ์ เป็นต้น

Multinomial Logistic Regression

ใช้เมื่อผลลัพธ์มีมากกว่า 2 คลาส เช่น ทำนายประเภทของสินค้าหรือความนิยม, จำแนกชนิดสัตว์, การพยากรณ์การเลือกซื้อของลูกค้า

Ordinal Logistic Regression

ใช้เมื่อผลลัพธ์มีลำดับชั้น (Ordinal) แต่ไม่ใช่ Continuous ซึ่งเป็นตัวแปรประเภท Category ที่เรียงลำดับได้ เช่น ความพึงพอใจ (น้อย, ปานกลาง, มาก) ระดับการศึกษา (ประถม, มัธยม, ปริญญาตรี, ปริญญาโท) และ ความรุนแรงของโรค (ระดับ 1, 2, 3) เป็นต้น

สูตรการคำนวณหลักของ Logistic Regression Model:

★ Py คือ ความน่าจะเป็น (probability) ของ class ใด ๆ

★ β คือ ค่าสัมประสิทธิ์ (coefficients) ของแต่ละ feature (ตัวแปร x) ช่วยบอกได้ว่า feature แต่ละตัวมีผลกระทบต่อการคาดการณ์มากน้อยแค่ไหน

จริง ๆ แล้ว Logistic Regression เป็นพื้นฐานของ Neural Network ถ้ามีหลาย logistic neuron มาต่อกัน ก็จะได้ neural network ชั้นแรก!

ใน Logistic Regression ผลลัพธ์จะได้ออกมาเป็นค่า p (probability) ระหว่าง 0 ถึง 1 เช่น

★ ตัวอย่างหนึ่ง อาจได้ p = 0.82 → "น่าจะใช่"

★ ตัวอย่างหนึ่ง อาจได้ p = 0.23 → "น่าจะไม่ใช่

แต่เรายังต้องตัดสินใจว่าจะต้อวจัดให้อยู่ใน class ไหน ทำให้โมเดล Logistic Regression จึงมีค่าที่เรียกว่า Threshold ในการช่วยตัดสินใจ (ส่วนมากมองว่าถ้ามากกว่าค่า Treshold ให้เป็น 1 และถ้าต่อกว่าให้เป็น 0)

โมเดลพื้นฐาน : Tree Based Model

โมเดลแบบต้นไม้ (Tree-Based Models) ใช้ โครงสร้างต้นไม้ตัดสินใจ (Decision Tree) เป็นพื้นฐาน โดยมีองค์ประกอบหลักดังนี้

1. Node

2. การแบ่ง (Splitting)

    ★ Split Rule คือ กฎที่ใช้แบ่งข้อมูลในแต่ละโหนด (เช่น X > 5 ลงไปยัง node A)
    ★ Split Criterion คือ เกณฑ์วัดประสิทธิภาพของการแบ่ง เช่น

3. การหยุดเติบโต (Stopping Criteria)

การกำหนดจุดสิ้นสุดของโมเดลสามารถทำได้หลายวิธี เช่น กำหนดความลึกสูงสุด (Max Depth) ของต้นไม้ หยุดแบ่งเองอัตโนมัติเมื่อจำนวนตัวอย่างในโหนดน้อยเกินไป (Min Samples Split) หรือหยุดการแบ่งเมื่อไม่มีการลด Impurity อย่างมีนัยสำคัญ เป็นต้น

Random Forest

โมเดล tree based model ที่เกิดจาก Dicision tree หลายโมเดลมารวมกัน หรือกล่าวให้ง่ายขึ้นคือ Dicision tree เปรียบเสมือนต้นไม้ 1 ต้น และ Random Forest คือการนำต้นไม่หลาย ๆ ต้นมารวมกัน โมเดลชนิดนี้สามารถใช้ได้ทั้ง Classification และ Regression เหมาะกับปัญหาที่ข้อมูลซับซ้อน มี feature interaction เยอะ และไม่แน่ใจว่าความสัมพันธ์เป็นเส้นตรงไหม

โมเดล Random Forest อาจมีปัญหาคือการขาดข้อมูลที่เพียงพอสำหรับต้นไม้แต่ละต้นในป่า ดังนั้น มันจึงต้องมีวิธีเตรียมข้อมูลของมันก่อนเข้าโมเดล เรียกว่า “Bootstrapping” ซึ่งเป็นการสุ่มหยิบข้อมูลบางส่วนจากข้อมูลต้นฉบับนำมาสร้างเป็นข้อมูลชุดใหม่ ๆ ที่เพียงพอสำหรับต้นไม้ทุกต้นในป่า ตามรูป

จากรูปจะเรียกว่าเป็นการทำ Bagging ใน Random Forest ซึ่งคือเทคนิคการรวมผลจากหลายโมเดลที่ฝึกจากชุดข้อมูลที่สุ่ม (Bootstrap Sampling) เพื่อเพิ่มความแม่นยำและลดความเสี่ยงของ overfitting ครับ

ประเภทแยกย่อย:

Random Forest Classifier

ใช้สำหรับจำแนกหมวดหมู่ เช่น ทำนายว่าลูกค้าจะ "ซื้อ" หรือ "ไม่ซื้อ", จำแนกชนิดของดอกไม้, ตรวจจับอีเมลสแปม

Random Forest Regressor

ใช้สำหรับพยากรณ์ค่าต่อเนื่อง เช่น ราคาบ้าน, รายได้ของบริษัทในอนาคต

Forest Classifier และ Logistic Regression มีจุดประสงค์ในการหา output แบบเดียวกันแต่มีโครงสร้างและการทำงานต่างกัน โดยเราแนะให้ใช้ Random Forest Classifier หากข้อมูลมีความซับซ้อนมาก (แต่แลกมากับการประมวลผลที่ช้ากว่า Logistic Regression

โมเดลพื้นฐาน : Neural Network

คือโมเดลคอมพิวเตอร์ที่เลียนแบบโครงสร้างการทำงานของสมองมนุษย์ ประกอบด้วยชุดของ "neurons" (โหนด) ที่เชื่อมต่อกันเป็น ชั้น (layers)

โมเดล Neural Network มักถูกนำมาใช้สำหรับเรียนรู้ข้อมูลที่มีรูปแบบซับซ้อน เช่น รูปภาพ ข้อความ หรือเสียง

โครงสร้างโดยพื้นฐานของโมเดล Neural Network ประกอบไปด้วย

1. Input Layer

รับข้อมูลเข้ามา (เช่น รูปภาพ, ตัวเลข)

2. Hidden Layers

ประมวลผลข้อมูลด้วยการคำนวณถ่วงน้ำหนัก + ผ่านฟังก์ชันแปลง (activation function)

3. Output Layers

ผลลัพธ์สุดท้ายจากการทำนาย

โดยการทำงานคร่าว ๆ ของโมเดลคือ เมื่อข้อมูลวิ่งเข้า network ผ่าน Input layer ทุกครั้งที่ผ่านออกไปชั้นอื่นๆ จะต้องมีการคำนวณน้ำหนัก (weights) และไบแอส (bias) ที่ neuron แต่ละตัวทุกครั้ง จากนั้นจึงใช้ activation function เพื่อแปลงข้อมูล และกระจายผลต่อไปยัง neuron ถัดไป สุดท้ายตอน train ใช้ Backpropagation ปรับ weights เพื่อให้โมเดลเก่งขึ้นเรื่อย ๆ

คำศัพท์น่ารู้

★ Weights

เป็นค่าที่วัดว่าข้อมูลสำคัญแค่ไหนในการส่งผ่านจาก neuron หนึ่งไปยัง neuron ถัดไประบบจะเรียนรู้การปรับ weights เพื่อให้ทำนายได้แม่นยำที่สุด

★ Bias

ค่าคงที่ที่ช่วยเลื่อนเส้นการตัดสินใจเพื่อเพิ่มความยืดหยุ่น เหมือนเป็นการตั้ง offset เพิ่มเติมไม่ให้อิงแค่ input อย่างเดียว

★ Activation function

หรือ ฟังก์ชันแปลง คือฟังก์ชันที่ทำให้ model เข้าใจความสัมพันธ์ที่ไม่ใช่เส้นตรง (non-linear) ได้ ยกตัวอย่างเช่น ReLU, Sigmoid

★ Backpropagation

Backpropagation หรือ กระบวนการย้อนกลับ มีหน้าที่ปรับ weights และ bias หลังจากคำนวณผลผิดพลาดหรือผลลัพธ์ออกมาไม่ได้ประสิทธิภาพพอ ทำให้ network เรียนรู้จากความผิดพลาดได้เรื่อย ๆ โดยใช้การหาค่าลงมาจากชั้น output → hidden layers → input (ย้อนกลับมา)

CNN (Convolutional Neural Network)

CNN คือ Neural Network ประเภทหนึ่งที่ถูกออกแบบมาเพื่อ วิเคราะห์ข้อมูลที่มีโครงสร้างเป็นตาราง เช่น รูปภาพ (2D pixel grid) หรือเสียง (waveform) ใช้การทำ Convolution เพื่อตรวจจับ คุณสมบัติลักษณะ (features) เช่น ขอบ (edges), รูปทรง (shapes) และพื้นผิว (textures) เหมาะสำหรับงาน เช่น Image Recognition, Object Detection, Medical Imaging เป็นต้น

Evaluation Metrics คือ ตัวชี้วัดที่ใช้ในการประเมินประสิทธิภาพของโมเดล Machine Learning ว่าโมเดลทำงานได้ดีแค่ไหน โดยใช้ข้อมูลที่เรารู้คำตอบจริงอยู่แล้ว เพื่อเปรียบเทียบกับผลลัพธ์ที่โมเดลทำนายออกมา

★ Negative คือ class รอง หรือก็คือ “ไม่สั่งของ No = Negative”

จะเห็นว่าด้านในตารางจะมีตัวอักษรย่ออยู่ 4 ตัวที่เราจะต้องทำความรู้จักนั้นคือ

จาก Class(Positive / Negative) ที่กล่าวมาข้างต้นก็คือส่วนประกอบของ Confusion Matrix หรือเรียกอีกอย่างว่า “ตารางแห่งความมึนงง” นั่นเอง ถ้าเราสามารถจำ Concept ได้ ก็จะทำให้เราอธิบายผลได้ดียิ่งขึ้นนั่นเอง

โดยเราสามารถคำนวณค่าสถิติหลายๆตัวจากตารางนี้ โดย Classification Metrics 5 ตัวที่นิยมใช้กัน คือ

วิธีพิจารณากราฟและประเมินผล AUC

     
  • AUC = 0.4 โมเดลแย่มาก (ทำนายผิดเยอะกว่าถูก)
  • AUC = 0.5 ทำนายได้ไม่ต่างจากการเดา (Random Guessing)
  • AUC = 0.7 โมเดลทำงานได้ดี ตามมาตรฐานทั่วไป
  • AUC = 0.9 โมเดลทำนายได้ดีมาก (แยกแยะ Positive/Negative ได้ดี)

Conclusion

โดยสรุปแล้ว Predictive Model หรือแบบจำลองการพยากรณ์ เป็นเครื่องมือสำคัญที่ช่วยให้เราสามารถวิเคราะห์และคาดการณ์เหตุการณ์ในอนาคตได้อย่างมีประสิทธิภาพ โดยอาศัยข้อมูลในอดีตมาประมวลผลเพื่อสร้างแบบจำลองที่มีความแม่นยำช่วยอำนวยความสะดวกให้แก่ชีวิตประจำวันหรืออุตสาหกรรมต่าง ๆ ได้มากมาย !

นอกจากนี้ ปัจจุบันการพัฒนาเทคโนโลยีด้านปัญญาประดิษฐ์ โดยเฉพาะ Large Language Models (LLMs) เช่น GPT Gemini หรือ BERT ยังมีบทบาทสำคัญในการสนับสนุนการทำ Predictive Model อีกด้วย ทั้งในด้านการสร้างฟีเจอร์จากข้อมูลไม่เป็นโครงสร้าง (Unstructured Data) อย่างข้อความ การสรุปผลเชิงสถิติจากข้อมูลจำนวนมาก ไปจนถึงการอธิบายผลลัพธ์ของแบบจำลองอย่างเข้าใจง่าย จึงกล่าวได้ว่า LLM เป็นอีกหนึ่งแรงผลักดันสำคัญที่ช่วยยกระดับความสามารถของ Predictive Modeling ให้มีประสิทธิภาพและเข้าถึงง่ายมากยิ่งขึ้น
